System Overview

This camera logistics system manages a multi-store franchise operation, tracking manufacturer relationships, parts supplier networks, inventory management, and store locations. The system enables complex queries across four interconnected entities to provide business intelligence and operational insights.

Entity Relationships Explained

Store (1:M) Camera

A single store location can stock and sell multiple camera models. Each store is identified by Store ID and maintains inventory across various camera products.

Camera (1:1) Manufacturer

Each specific camera model is assembled by exactly one manufacturer. The manufacturer relationship is critical for tracking product sourcing and assembly location.

Manufacturer (1:M) Camera

A manufacturer can assemble and produce multiple different camera models, enabling efficient supply chain management and diversified product offerings.

Camera (1:1) Parts Supplier

Each camera model receives specialized parts from exactly one parts supplier that specializes in components for that particular camera.

Parts Supplier (1:M) Camera

A single parts supplier can provide specialized components to multiple camera models, creating economies of scale in parts manufacturing.

Database Tables & Attributes

Store Table
  • PK: store_ID - Primary Key
  • store_name - Franchise name (shared across locations)
  • store_address - Physical address of store location
  • store_city - City where store is located
Camera Table
  • PK: camera_ID - Primary Key
  • camera_name - Product name/model
  • camera_price - Retail price
  • camera_quantity - Inventory quantity
  • FK: store_ID - References Store table
  • FK: manufacturer_ID - References Manufacturer table
  • FK: supplier_ID - References Parts Supplier table
Manufacturer Table
  • PK: manufacturer_ID - Primary Key
  • brand_name - Brand/manufacturer name
  • manufacturer_country - Country of origin
  • FK: supplier_ID - References Parts Supplier table
Parts Supplier Table
  • PK: supplier_ID - Primary Key
  • supplier_name - Name of parts supplier
  • supplier_country - Country of origin
  • supplier_phone - Contact phone number

Advanced SQL Queries

The system implements 10 sophisticated queries demonstrating JOINs, WHERE clauses, aggregation functions, and complex filtering logic:

1. Store Locations by City

Find all store names and addresses in a specific city

SELECT store_name, store_address FROM store WHERE store_city = 'Carson';
2. Manufacturer Inventory Analytics

Calculate total quantity and average price for cameras from specific manufacturer

SELECT manufacturer_id, SUM(camera_quantity) as total_quantity, AVG(camera_price) as average_price FROM camera WHERE camera.manufacturer_id = '9999';
3. Supplier Location Filtering

Find all parts suppliers in a specific country

SELECT supplier_name FROM partsupplier WHERE supplier_country = 'USA';
4. Multi-Table JOIN - Manufacturer Availability

Find manufacturers with inventory in stores at specific location

SELECT manufacturer.brand_name, manufacturer.manufacturer_country FROM manufacturer JOIN camera ON manufacturer.manufacturer_id = camera.manufacturer_id JOIN store ON camera.store_id = store.store_id WHERE store.store_city = 'San Pedro';
5. Supplier-Store Relationship

Identify which suppliers provide cameras to specific store

SELECT partsupplier.supplier_name FROM partsupplier JOIN manufacturer ON partsupplier.supplier_id = manufacturer.supplier_id JOIN camera ON manufacturer.manufacturer_id = camera.manufacturer_id JOIN store ON camera.store_id = store.store_id WHERE store.store_id = '77';
6. Inventory Threshold Query

Find stores with inventory above specific quantity threshold

SELECT store_name, store_city FROM store JOIN camera ON store.store_id = camera.store_id WHERE camera_quantity >= 5;
7. Product Availability by Store

Locate stores selling specific camera model

SELECT store.store_name, store.store_address FROM store JOIN camera ON store.store_id = camera.store_id WHERE camera.camera_name = 'Hero 9';
8. Multi-City Inventory Search

Get inventory details for cameras in multiple cities

SELECT camera.camera_quantity, camera.camera_name, camera.camera_price FROM camera JOIN store ON camera.store_id = store.store_id WHERE store.store_city = 'Oxnard' OR store.store_city = 'Carson';
9. Supplier Product Distribution

Track specific supplier's products across all stores

SELECT camera.camera_name, camera.camera_price, camera.camera_quantity FROM camera JOIN partsupplier ON camera.supplier_id = partsupplier.supplier_id JOIN store ON camera.store_id = store.store_id WHERE partsupplier.supplier_name = 'Edmund Optics';
10. Domestic Business Partnerships

Find manufacturers and suppliers based in same country doing business together

SELECT manufacturer.brand_name AS 'American Manufacturer', partsupplier.supplier_name AS 'American Supplier' FROM manufacturer JOIN partsupplier ON manufacturer.supplier_id = partsupplier.supplier_id WHERE manufacturer.manufacturer_country = 'USA' AND partsupplier.supplier_country = 'USA';

Java Swing GUI Application - JDBC Integration

Interactive Search Interface

Developed a user-friendly Java Swing application enabling real-time database queries through a GUI search interface. Users can search for cameras by name or by store city without writing SQL code.

Dynamic Results Table

Implemented dynamic JTable with JScrollPane for displaying query results. The table shows camera name, price, quantity, store city, and store address in a formatted, easy-to-read layout.

JDBC Connection Management

Established secure MySQL connection using JDBC with proper credential management and connection lifecycle handling. Implemented proper resource cleanup with connection.close() to prevent memory leaks.

Prepared Statements & SQL Injection Prevention

Utilized PreparedStatement with parameterized queries to prevent SQL injection attacks. Search parameters are safely bound to the query before execution.
